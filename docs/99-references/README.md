# References

## Description

Curated official documentation and trusted links for DevOps, Linux, full stack development, and operations topics.

## Common Commands

```bash
man command_name
command_name --help
info command_name
```

## Examples

Open a local manual page:

```bash
man systemctl
```

Check command help:

```bash
docker --help
```

## Troubleshooting

- Prefer official documentation when commands change across versions.
- Record the date or tool version when a note depends on a specific release.
- Avoid copying long external articles; summarize and link to the source.
